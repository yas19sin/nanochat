# FineWeb-Edu Darija Translated

Moroccan Darija (الدارجة المغربية) translation of a 2B-token slice of
[HuggingFaceFW/fineweb-edu](https://huggingface.co/datasets/HuggingFaceFW/fineweb-edu),
produced with [`Lyte/tiny-aya-darija-v5`](https://huggingface.co/Lyte/tiny-aya-darija-v5)
(3.35B-param Cohere2 model, bf16).

The goal is to give Darija-focused pretraining runs access to the same curated,
educational web text that powers high-quality English base models — translated
naturally into Moroccan Darija with code-switched technical terms preserved.

## Dataset at a glance

| Field            | Value                                   |
|------------------|-----------------------------------------|
| Source           | `HuggingFaceFW/fineweb-edu` (sample-10BT) |
| Translator       | `Lyte/tiny-aya-darija-v5` (bf16)        |
| Target language  | Moroccan Darija (`ary`), Arabic script   |
| Format           | Parquet shards (~10k rows each, ~19 MB)  |
| Rows             | ~938k and growing                        |
| Output tokens    | Target ~2 B                              |
| License          | ODC-By 1.0 (inherited from fineweb-edu)  |

## Schema

Each row contains:

| Column           | Type           | Description                                    |
|------------------|----------------|------------------------------------------------|
| `src_idx`        | `int64`        | Row index in the source fineweb-edu stream     |
| `en`             | `large_string` | Original English text (truncated to 1500 chars)|
| `darija`         | `large_string` | Moroccan Darija translation                    |
| `output_tokens`  | `int64`        | Number of tokens generated by the translator   |

## Translation pipeline

- **Model:** `Lyte/tiny-aya-darija-v5` served with `vllm==0.11.0` on an
  NVIDIA RTX 5090 (Triton attention backend).
- **Sampling:** `temperature=0.3, top_p=0.98, top_k=300,
  repetition_penalty=1.15, max_tokens=512`.
- **Prompt:** single-turn chat template with system message
  *"Translate the following English text into Moroccan Darija (الدارجة المغربية).
  Output ONLY the Darija translation."*
- **Throughput:** ~4.5–6.9 k output tokens/s per GPU (bs=384, seq 4096).
- **Sharding:** parquet shards of 10 000 accepted rows, uploaded to the Hub
  incrementally; the run is resumable via `manifest.json`.

Code: [`scripts/translate_vllm_darija.py`](https://github.com/Lyte-org/nanochat/blob/main/scripts/translate_vllm_darija.py)

## Quality filters

A row is kept only if *all* of the following hold for its Darija output:

1. Length ≥ 20 characters.
2. Length ratio `len(darija) / len(en)` between **0.4** and **3.0**.
3. At least **10 Arabic-script characters** (filters out latinized Darija /
   empty / English echoes).

Rows failing any check are dropped silently; `src_idx` therefore has gaps.

## What the translations look like

The translator tends to:

- Stay in natural Moroccan Darija register
  (`كيفما`, `بحال`, `دياله/ديالها`, `كيكون`, `واخا`, `على حساب`).
- Keep technical terms in Latin script when helpful
  (`QuickTime`, `Hypotheneumus hampei`, `stormwater runoff`, `PDF`, `font`).
- Preserve paragraph structure and lists.
- Occasionally drift on numeric details or rare named entities
  (e.g. `115 → 155`, `1776 → 1726`, *Holm oak → البلوط الفليني* which is
  actually cork oak). Mitigate with a 50/50 English + Darija mix at
  pretraining or with downstream fact-grounded supervision.

A small side-by-side comparison between the bf16 translator and its Q8\_0
quantization can be found in the source repo at
[`dev/q8_vs_bf16_probe.md`](https://github.com/Lyte-org/nanochat/blob/main/dev/q8_vs_bf16_probe.md).

## Intended use

- Pretraining / continued-pretraining corpus for Moroccan Darija LLMs.
- Building paired EN↔DARIJA data for machine translation.
- Darija instruction tuning when combined with human-curated SFT data.

**Not intended for** retrieving factual claims verbatim from the `darija`
field — cross-check against `en` first.

## Limitations & biases

- **Machine-translated only.** No human review. Expect the full spectrum of
  MT artefacts: number drift, mistranslated named entities, occasional
  register slips, and rare hallucinations.
- **Educational web bias.** Inherits fineweb-edu's lean toward formal,
  US-English, STEM-heavy content. Casual everyday Darija is underrepresented.
- **Script bias.** Latin-script Darija is filtered out, so the dataset does
  not reflect how Darija is actually written on social media.
- **Model-specific vocabulary.** Stylistic choices reflect
  `Lyte/tiny-aya-darija-v5`'s training distribution and may over-use certain
  connectives.
